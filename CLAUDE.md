# pg_timers — Agent Guide

## What this is
A PostgreSQL C extension that provides precise, low-latency timer scheduling via a background worker. Distributed as a standard PostgreSQL extension (PGXN) and as container images (GHCR).

## Tooling split

| Concern | Tool | Why |
|---|---|---|
| Dev shell / editor / LSP | Nix (`nix develop`) | Native toolchain, instant, no Docker needed |
| Local compilation | `make USE_PGXS=1` inside `nix develop` | Fast iteration without containers |
| Dev postgres | `make dev` (docker compose) | ABI-consistent with what ships |
| Tests | `make test` (docker compose) | Same environment as CI and release image |
| Release image | `Dockerfile.release` (multi-stage) | Official `postgres` base, full DX for users |
| CNPG image | `Dockerfile.cnpg` (multi-stage) | Official `postgres` base, ABI-compatible |
| CI tests | `docker compose --profile test run --rm test` | No Nix on CI runners |
| Release publishing | `docker buildx` + QEMU | Multi-arch (amd64 + arm64) via GHCR |

## Commands

```bash
make dev      # start dev postgres on port 5433 (docker compose)
make test     # run pgTAP suite (docker compose)
make psql     # open psql in running dev container
make down     # stop and remove containers + volumes
```

## Rules for agents

- **Never install anything locally.** All runtime tasks go through Docker.
- **Never touch `flake.nix`** for runtime concerns. The flake provides dev shells and local extension builds only — it is not used for tests, dev postgres, or image builds.
- **Never use `nix run`** for anything. Tests and dev postgres run via `docker compose`.
- **Tests run via docker compose**, not natively. Use `make test` or `PG_MAJOR=<ver> docker compose --profile test run --rm test`.
- **Version lives in `pg_timers.control`** (`default_version`). release-please owns it. Never edit it manually or maintain a separate version file.
- **Migrations are append-only** under `sql/migrations/`. The versioned install file `sql/pg_timers--<version>.sql` is generated by `make` from all migrations concatenated — do not edit it directly.
- **The Makefile** is required by PGXS and by the Dockerfiles. Do not remove it or its PGXS variables.

## Project layout

```
pg_timers/
├── flake.nix                  # Nix: devShells + local extension builds only
├── Makefile                   # PGXS build definition (used by Docker and nix build)
├── pg_timers.control          # Extension metadata — single source of version truth
├── META.json                  # PGXN metadata
├── Dockerfile                 # Dev/test image (postgres + pgTAP + extension)
├── Dockerfile.release         # Release image (multi-stage, official postgres base)
├── Dockerfile.cnpg            # CNPG image (multi-stage, scratch final stage)
├── docker-compose.yml         # Dev and test services
├── src/                       # C source (pg_timers.c, bgworker.c, functions.c, pg_timers.h)
├── sql/migrations/            # Append-only SQL migrations
├── t/                         # pgTAP tests (*.sql)
├── docker-entrypoint-initdb.d/  # Init scripts run by postgres on first start (dev image only)
├── examples/                  # CNPG cluster example
└── .github/workflows/
    ├── ci.yml                 # Test matrix (pg 15–18) via docker compose
    └── release.yml            # release-please → docker buildx → GHCR
```
